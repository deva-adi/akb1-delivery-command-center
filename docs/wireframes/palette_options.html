Claude's recommendation

Option D (Executive Mid) is my primary recommendation. It reduces the darkness you asked to mitigate without losing the premium analytics-tool feel. Sits between Option A (current, too dark) and Option E (full light). Works well for both the laptop screen and projected screen in a steering committee room. Handles dark-mode follower on system preference gracefully via CSS variable swap.

Option E (Executive Light) is the second recommendation if you prefer full light mode. McKinsey, BCG, Bain, Deloitte use light as the default for boardroom analytics. Prints cleanly. Screenshots transfer to client slides without a post-process. Gold-accent pops harder.

Best of both worlds: I can implement Option D as default with Option E available as a user-toggle in the role badge area. CSS variables make this a 2-line toggle in production. One wireframe build, two palette files.
